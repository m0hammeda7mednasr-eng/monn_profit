# Implementation Plan: User Data Isolation System

## Overview

This implementation plan breaks down the user data isolation feature into discrete, testable coding tasks. The approach follows a layered architecture: first establishing backend infrastructure (authentication middleware and data filtering), then refactoring route handlers to use the centralized logic, updating frontend UI for role-based visibility, and finally comprehensive testing including property-based tests.

Each task builds incrementally on previous work, with checkpoints to validate functionality before proceeding. The implementation ensures employees see only their own data while admins have full visibility, and all users share access to Shopify-synced resources.

## Tasks

- [ ] 1. Create backend authentication middleware
  - [x] 1.1 Implement centralized JWT validation middleware
    - Create `backend/src/middleware/auth.js` file
    - Implement `authenticateToken` function that extracts JWT from Authorization header
    - Validate token using JWT_SECRET and handle errors (missing, invalid, expired tokens)
    - Attach decoded user object to `req.user` with fields: id, email, role
    - Set convenience flag `req.user.isAdmin = (req.user.role === 'admin')`
    - Return 401 Unauthorized for missing/invalid tokens with appropriate error messages
    - _Requirements: 7.1, 7.2, 7.3, 7.4, 7.5_

  - [ ]\* 1.2 Write unit tests for authentication middleware
    - Test valid JWT token with admin role → req.user populated correctly
    - Test valid JWT token with employee role → req.user populated correctly
    - Test missing JWT token → 401 Unauthorized
    - Test invalid JWT token → 401 Unauthorized
    - Test expired JWT token → 401 Unauthorized
    - Test malformed Authorization header → 401 Unauthorized
    - _Requirements: 7.1, 7.2, 7.3, 7.4_
